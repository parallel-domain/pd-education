Parking Notebook¶

In this notebook, we'll explore setting up parking scenarios to capture data from. To do this, we'll take the usual approach of loading a map and populating it with agents, but we will learn more about using the LaneSpawnPolicy we pass to the ego vehicle's PositionRequest to control the type of scene we end up with. In this case, we'll request that the ego's position is for the ParkingSpace lane type, which will lead to a parking-centric scene, as all of the LocationRelativePositionRequest we use for the other agents will follow the ego vehicle placement. We will also see how to modify the parking environment passing EnvironmentParameters into the scenario's set_environment method.

2. Create a Scenario Building Function ¶

Since we'll want to create multiple similar scenarios to explore the features available in parking scenes, it is helpful first define a function so we can easily recreate our base scenario in later cells with a simple function call.

NOTE the use of the ParkingSpace lane type in the ego agent's lane spawn policy, which establishes the parking nature of the scenarios we generate using this function.

In [2]:
def build_parking_scenario(location_name: str = "SJ_237AndGreatAmerica", seed: int = 12345):

    # Create sensor rig
    sensor_rig = data_lab.SensorRig().add_camera(
        name="Front",
        width=1920,
        height=1080,
        field_of_view_degrees=70,
        pose=Transformation.from_euler_angles(
            angles=[0.0, 0.0, 0.0], order="xyz", degrees=True, translation=[0.0, 0.0, 1.5]
        ),
        annotation_types=[Depth, SemanticSegmentation2D, InstanceSegmentation2D],
    )

    # Create scenario
    scenario = data_lab.Scenario(sensor_rig=sensor_rig)
    scenario.random_seed = seed

    # Set location
    scenario.set_location(data_lab.Location(name=location_name, version=LOCATION_VERSION))

    # Set weather variables and time of day
    scenario.environment.time_of_day.set_category_weight(data_lab.TimeOfDays.Day, 1.0)
    scenario.environment.clouds.set_uniform_distribution(min_value=0.0, max_value=0.2)
    scenario.environment.rain.set_constant_value(0.0)
    scenario.environment.fog.set_constant_value(0.0)
    scenario.environment.wetness.set_uniform_distribution(min_value=0.1, max_value=0.3)

    # Place ego in the world - *THIS IS WHERE WE DEFINE THE REQUESTED LANE TYPE FOR EGO SPAWNING*
    scenario.add_ego(
        generator=EgoAgentGeneratorParameters(
            agent_type=AgentType.VEHICLE,
            vehicle_spawn_data=VehicleSpawnData(
                vehicle_behavior=VehicleBehavior(
                    parking_scenario_goal=PositionRequest(
                        lane_spawn_policy=LaneSpawnPolicy(lane_type=EnumDistribution(probabilities={"ParkingSpace": 1.0})),
                        yaw_offset=CenterSpreadConfig(center=0, spread=0),
                    ),
                )
            ),
        )
    )

    # Place other traffic agents
    scenario.add_agents(
        generator=ParkedVehicleGeneratorParameters(
            spawn_probability=CenterSpreadConfig(center=0.6),
            position_request=PositionRequest(
                location_relative_position_request=LocationRelativePositionRequest(
                    agent_tags=["EGO"],
                    max_spawn_radius=100.0,
                )
            ),
        )
    )

    return scenario

Specify the type of parking spaces which should appear in the scenario¶

In this case we are requesting 45 degree parking spaces marked with single white lines.

In [5]:
parking_space_type = 'ANGLE_45'
demarcation_type = {"SINGLE": 1, "DASHED": 0, "DOUBLE_OPEN": 0, "DOUBLE_SQUARED": 0, "DOUBLE_ROUND": 0}
line_color_rgb = Float3(x=1, y=1, z=1)

scenario.set_environment(
    EnvironmentParameters(
        parking_space_data=ParkingSpaceData(
            parking_lot_angle_distribution=EnumDistribution(probabilities={"ANGLE_45": 1.0}),
            lot_parking_delineation_type=EnumDistribution(
                probabilities=demarcation_type
            ),
            delineation_color=[line_color_rgb],
            delineation_wear_amount=CenterSpreadConfig(center=0, spread=0),
            parking_space_tint=[Float3(x=1, y=1, z=1)],
            parking_space_grunge_amount=CenterSpreadConfig(center=0, spread=0),
        )
    )
)

Render Scenario 1 ¶

In this example, we just want to get a preview of our scenario without saving it to disk. To do this, we can use the preview_scenario method. Since we just want to get a look at the scene and the parking maneuver, we can avoid a longer wait time by rendering at longer capture rate. Since our simulation is at 100Hz, setting the sim_capture_rate to 100 will give us data capture at 1Hz, which is appropriate for a preview. This way we can see a 10 second scene with only 10 frames_per_scene.

In [6]:
data_lab.preview_scenario(
    scenario=scenario,
    frames_per_scene=10,
    sim_capture_rate=100,
    sim_instance=SimulationInstance(name="fsbapfxq"),
    render_instance=RenderInstance(name="fsbapfxq"),
)

parking_space_type = 'PERPENDICULAR'

scenario.set_environment(
    EnvironmentParameters(
        parking_space_data=ParkingSpaceData(
            parking_lot_angle_distribution=EnumDistribution(probabilities={parking_space_type: 1.0}),
            lot_parking_delineation_type=EnumDistribution(
                probabilities=demarcation_type
            ),
            delineation_color=[line_color_rgb],
            delineation_wear_amount=CenterSpreadConfig(center=0.5, spread=0),
            parking_space_tint=[Float3(x=1, y=1, z=1)],
            parking_space_grunge_amount=CenterSpreadConfig(center=0, spread=0),
        )
    )
)
